# Reproducibility documentation

**Manuscript:** *Deep reinforcement learning for simulated autonomous heat
stress mitigation in dairy cattle.*

This document answers the reviewer's reproducibility requests and fills every
"should be inserted from the final code logs" placeholder. **No number in
Table 3 or Table 4 of the manuscript is changed.** The published results are
preserved as the *primary run*; the seeded multi-seed runs and the
reward-weight sensitivity analysis below are *supplementary confirmations*.

---

## 1. Code that produced the published results

| Artefact | File |
|---|---|
| Environment, agents, networks, reward, data loader | `streamlit_dqn.py` |
| Training/evaluation driver (Table 3, Figures 1–4) | `run_experiment.py` |
| Published evaluation numbers (Table 3) | `results/paper_figures/results.json` |

`results/paper_figures/results.json` matches Table 3 to four decimal places
(e.g. Double DQN reward 18.2559 → 18.26 ± 3.89), confirming this is the exact
code path behind the article.

## 2. Random seeds

* **Primary published run:** executed **without** a global seed
  (`run_experiment.py` sets no `np.random.seed` / `torch.manual_seed`). The
  reported Table 3/4 numbers therefore come from one specific stochastic run.
* **Reproduction:** `reproducibility/reproduce.py` sets a global seed
  (`random`, `numpy`, `torch`) and was run over **5 independent seeds:
  42, 123, 456, 789, 1011**.

## 3. Train / validation / test split

The environment (`HeatStressEnv.reset()`) samples each episode by choosing a
random cow and a random 24-hour (48-step) start window from the **full
preprocessed dataset** (6,607 samples, 10 cows). **There is no separate
temporal hold-out**: training and evaluation episodes are drawn from the same
preprocessed pool. The 50 evaluation episodes use the trained policy in
greedy mode (`training=False`). This is documented truthfully in the manuscript
Methods (the corresponding text changes are kept with the manuscript, not in this repository).

## 4. Number of repeated independent runs

* Primary: **1** run (published).
* Supplementary reproduction: **5** seeds (Section 7).
* Sensitivity analysis: **8** reward-weight settings, fixed seed 42 (Section 8).

## 6. Dataset & missing-data accounting

* Complete cow-time grid: 10 cows × 14 days × 48 intervals = **6,720** potential
  observations. **6,607 retained (98.3 %)**, 113 excluded (1.7 %) — matches the
  manuscript.
* Loader reproduces the published dataset statistics exactly: THI 58.6–82.7,
  mean THI 71.2, heat-stress ratio (THI > 72) = 47.6 %.
* **Behavioural-modality note (important):** in `MmCowsDataLoader.load()` the
  `lying_ratio` and `activity` state features are set to fixed
  population-level constants (0.5 and 0.3); CBT, THI and milk yield are loaded
  from the real MmCows streams. The behavioural inputs are therefore constant
  priors in this proof of concept, not per-animal IMU time series. This is
  stated in the manuscript Limitations.

## 7. Supplementary: repeated-run confirmation (5 seeds)

Mean ± SD across seeds {42, 123, 456, 789, 1011}. Generated by
`reproduce.py`; raw per-seed files `results/reproducibility/results_seed*.json`.

| Method | Reward (mean ± SD) | No-intervention (action 0) |
|---|---|---|
| No Action (baseline) | 17.88 ± 0.60 | — |
| THI Threshold (baseline) | 13.71 ± 0.49 | — |
| CBT Threshold (baseline) | 17.47 ± 0.44 | — |
| Random (baseline) | 13.49 ± 0.57 | — |
| Standard DQN | 17.98 ± 0.56 | 100.0 % |
| **Double DQN** | **17.76 ± 1.35** | 84.7 % ± 21.7 |
| Dueling DQN | 18.16 ± 0.44 | 100.0 % |
| Double Dueling DQN + PER | 16.92 ± 0.79 | 99.8 % ± 0.5 |

**Conclusion (consistent with the published single run):** the learned agents
cluster around the No-Action baseline and overwhelmingly select
no-intervention, while clearly beating the high-energy THI-Threshold and Random
baselines. The qualitative findings of the paper are reproduced across seeds.

## 8. Supplementary: reward-weight sensitivity analysis

Double DQN re-trained (seed 42) under 8 weight triplets (α·milk + β·comfort −
γ·energy, each summing to 1.0). Full table: `sensitivity_reward_weights.csv`.

| α (milk) | β (comfort) | γ (energy) | Reward | No-intervention % |
|---|---|---|---|---|
| 0.40 | 0.40 | 0.20 *(published)* | 18.54 | 100.0 |
| 0.45 | 0.45 | 0.10 | 20.43 | 100.0 |
| 0.50 | 0.40 | 0.10 | 18.22 | 100.0 |
| 0.34 | 0.33 | 0.33 | 15.86 | 100.0 |
| 0.30 | 0.30 | 0.40 | 14.76 | 100.0 |
| 0.25 | 0.25 | 0.50 | 12.87 | 100.0 |
| 0.30 | 0.50 | 0.20 | 22.96 | 100.0 |
| 0.20 | 0.60 | 0.20 | 27.38 | 100.0 |

**Conclusion:** the no-intervention policy is **robust to the reward weights**
— even when the energy penalty is halved or comfort is weighted up to 0.6, the
agent still withholds cooling in 100 % of evaluation steps. The convergence to
no-intervention is therefore driven by the moderate thermal load of the
dataset, not by the particular weight choice. This strengthens the
Discussion's existing argument.

## 10. How to reproduce

```bash
cd paper_a_rl_heat_stress
python -m venv venv && source venv/bin/activate
pip install -r reproducibility/requirements.txt

# 1) seeded multi-seed confirmation (writes results + models + manifest)
python reproducibility/reproduce.py --seeds 42 123 456 789 1011

# 2) reward-weight sensitivity
python reproducibility/sensitivity_reward_weights.py --seed 42

# 3) re-rendered figures
python reproducibility/make_figures.py
python reproducibility/make_fig2_training.py
```

## 11. Saved model checkpoints

`reproduce.py` saves every trained network to
`results/reproducibility/models/<agent>_seed<N>.pt` (PyTorch state dicts via
`DQNAgent.save`). Note: the **original** Table 3 networks were not checkpointed
by `run_experiment.py`; the saved checkpoints are from the seeded reproduction.
